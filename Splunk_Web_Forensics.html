1. Lab Overview & Objectives

Purpose

To investigate suspicious web activity involving long HTTP requests containing Base64-encoded payloads, correlate web-layer exploitation with host-level execution, and determine the scope of compromise using Splunk.

Objectives

  • Detect malicious web requests in Apache access and error logs.
  • Confirm backend execution through Sysmon telemetry.
  • Extract and decode Base64-encoded attacker payloads.
  • Reconstruct the attacker’s execution and reconnaissance chain.

2. Investigation Methodology

2.1 Suspicious Web Request Detection

index=windows_apache_access (cmd.exe OR powershell OR Invoke-Expression) | table _time host clientip uri_path uri_query status

Multiple requests were identified containing encoded PowerShell commands passed via HTTP parameters, indicating attempted command injection through the web application.

2.2 Apache Error Log Validation

index=windows_apache_error ("cmd.exe" OR "powershell" OR "Internal Server Error")

HTTP 500 responses confirmed that attacker input reached backend execution logic, supporting successful exploitation attempts rather than blocked probes.

2.3 Host-Level Process Correlation

index=windows_sysmon ParentImage="*httpd.exe*"

Sysmon logs showed Apache spawning cmd.exe and powershell.exe, which is anomalous behavior for a web server and confirms OS-level command execution.

2.4 Post-Exploitation Reconnaissance

index=windows_sysmon *cmd.exe* *whoami*

Execution of whoami.exe was observed, indicating attacker reconnaissance to identify execution context after successful injection.

2.5 Encoded Payload Assessment

index=windows_sysmon Image="*powershell.exe*" (CommandLine="*EncodedCommand*" OR CommandLine="*enc*")

Encoded PowerShell payloads were identified and decoded. No evidence indicated successful execution beyond reconnaissance activity.

5. Conclusions

This investigation confirmed a successful web-based command injection that resulted in OS-level execution and attacker reconnaissance. Splunk correlation across Apache and Sysmon telemetry enabled accurate validation of impact and scope. No persistence or lateral movement was observed during the analyzed timeframe.

6. Recommendations

  • Implement WAF rules for command injection and encoded payload detection.
  • Alert on Apache spawning system interpreters.
  • Log and monitor PowerShell encoded command usage.
  • Apply strict input validation on CGI and web endpoints.
